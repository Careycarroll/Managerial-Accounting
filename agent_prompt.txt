# Managerial Accounting Interactive -- Agent Prompt

You are a code agent for the Managerial Accounting Interactive project. This is a Vite-based multi-page web application that teaches Horngren's Cost Accounting (17th Edition) through interactive tools.

---

## Project Context

Goal: Build a comprehensive interactive learning companion covering all 24 chapters of Horngren's Cost Accounting. Students learn concepts chapter by chapter (Learn section), then apply them to real business decisions (Apply section).

Live site: https://careycarroll.github.io/Managerial-Accounting/
Repo: ~/Github Projects/Managerial-Accounting/
Stack: Vite 7, vanilla JS (ES modules), CSS custom properties, KaTeX, vite-plugin-pwa

## Architecture

### Two Sections
- Learn -- 24 chapter pages (pages/learn/ch01.html through ch24.html). Each page: hero, learning objectives, 3-5 interactive tools, show-work panels, key terms panel, chapter complete marker.
- Apply -- 12 scenario pages (pages/apply/*.html). Each scenario has three depth levels: Concept / Analysis / Simulation.

### File Structure

    pages/learn/          One HTML per chapter
    pages/apply/          One HTML per scenario
    js/core/              main.js, progress-tracker.js
    js/components/        randomizer.js, journal-entry.js, show-work.js (all built)
    js/charts/            chart-core.js (built), cvp-chart.js (built)
    js/engine/            scenario-engine.js + scenarios/*.json (not yet built)
    js/learn/             index.js, ch01.js, ch02.js, ch03.js, ch04.js (all built)
    js/apply/             Not yet built
    js/data/              managerial-terms.js (not yet built)
    css/base.css          Full custom property system (built)
    css/components.css    Shared UI components (built)
    css/learn.css         Learn section styles (built)
    css/apply.css         Not yet built

### Key Patterns
- Vite multi-page -- vite.config.js auto-discovers all HTML files in pages/learn/ and pages/apply/
- Base path -- process.env.NODE_ENV === 'production' ? '/Managerial-Accounting/' : '/'
- Progress tracking -- js/core/progress-tracker.js uses localStorage. Import markChapterComplete, isChapterComplete, resetChapter, getProgress.
- Randomizer -- js/components/randomizer.js. Call initRandomizer(buttonId, fields, onRandomize?) on any tool with numeric inputs.
- Show Work -- js/components/show-work.js. Call renderShowWork(containerEl, steps, options) after any calculation.
- Journal Entry -- js/components/journal-entry.js. Call renderJournalEntry(containerEl, entries, options).
- Canvas charts -- chart-core.js provides base Chart class. cvp-chart.js extends it. All charts: crosshair, tooltip, click-to-pin, scroll-to-zoom, drag-to-pan, double-click-to-reset.
- Full-bleed layout --.full-bleed class in components.css for elements that need to break out of container--tool width.
- KaTeX -- installed as a dependency for formula rendering across chapter tools.

### CSS Design System
All custom properties defined in css/base.css:
- Colors: --color-primary (#1a365d navy), --color-accent (#c8973a gold), semantic colors (success/warning/danger/info), gray scale
- Depth colors: --color-concept (blue), --color-analysis (green), --color-simulation (purple)
- Spacing: --space-1 through --space-16
- Typography: --font-sans (Inter), --font-mono (JetBrains Mono), --font-size-xs through --font-size-4xl
- Shadows, radii, transitions all defined
- Max widths use min() for responsive scaling: min(900px, 92vw) for tools, min(1200px, 92vw) for content

## Learn Page Template

Every chapter page follows this structure. Use Ch. 4 as the reference implementation.

HTML structure:
    Hero with chapter nav, badge, title, progress indicator
    Learning Objectives card
    Tool 1 of N... Tool N of N (alternating learn-section / learn-section--alt)
    Key Terms grid
    Chapter Complete card with mark-complete button and next chapter link
    script type=module src=/js/learn/chXX.js

HTML rules:
    No placeholder output divs in HTML. JS creates and appends all output elements dynamically.
    Only inputs, labels, buttons, and static structure belong in HTML.
    Output divs created in JS using getOrCreate(id, tag, className, parent) pattern.

JS file pattern:
    import { markChapterComplete, isChapterComplete, resetChapter } from '/js/core/progress-tracker.js';
    import { initRandomizer } from '/js/components/randomizer.js';
    import { renderShowWork } from '/js/components/show-work.js';
    DATA constants at top
    const fmt = n => '$' + Math.round(n).toLocaleString();
    const fmtPct = n => (n * 100).toFixed(1) + '%';
    function getOrCreate(id, tag, className, parent) -- helper for dynamic output elements
    One init function per tool
    Every numeric tool gets a randomizer button and a show-work panel
    initChapterComplete() wires mark-complete button and injects reset button after completion
    document.addEventListener('DOMContentLoaded', () => { init all tools });

## Component Reference

### initRandomizer(buttonId, fields, onRandomize?)
    fields: [{ id, min, max, step, integer?, constraint? }]
    constraint: 'lessThan:otherId' or 'greaterThan:otherId'
    onRandomize: callback fired after all fields are set (use to trigger calculate)

### renderShowWork(containerEl, steps, options?)
    steps: [{ label, formula, values, result, highlight?, note?, step? }]
    options: { title, defaultOpen, id }
    highlight: true = blue background, marks key result
    Always call after output is rendered, never before

### renderJournalEntry(containerEl, entries, options?)
    entries: [{ account, debit, credit, indent }]
    options: { title, subtitle, showTotals, balanced }

### CVPChart(canvasEl, config)
    config: { sellingPrice, variableCost, fixedCosts, maxUnits, currentUnits? }
    chart.update(config) -- redraws with new config, resets zoom and pins

### getOrCreate(id, tag, className, parent)
    const el = document.getElementById(id) || (() => {
      const e = document.createElement(tag || 'div');
      e.id = id;
      if (className) e.className = className;
      if (parent) parent.appendChild(e);
      return e;
    })();

## Development Rules

Discovery first. Before touching unfamiliar files, run discovery commands to map current state. Never assume file contents.

Always audit before building. Clear audit.txt first:
    > audit.txt && { commands; } >> audit.txt && echo audit.txt written

File creation. Always use python3 for all file writes. Never use heredocs for JS or CSS -- they break on backticks and special characters.

Terminal commands only. All changes delivered as executable zsh commands.

No inline comments in zsh. Do not put # comment lines inside command blocks.

No placeholder divs in HTML. JS creates all output elements dynamically after user action.

ls -R works. find does not. Use ls -R for directory listing.

Kill dev server:
    lsof -ti :5173,:5174,:5175,:5176 | xargs kill -9 2>/dev/null; pkill -9 -f vite 2>/dev/null; echo done

Commit after each chapter. One commit per completed chapter page. Message format:
    feat: Chapter N -- Title
    - pages/learn/chNN.html: what is in it
    - js/learn/chNN.js: tools built
    - css changes if any

Python3 for all file verification:
    python3 -c "
    import os
    path = os.path.expanduser('~/Github Projects/Managerial-Accounting/file.js')
    with open(path) as f: lines = f.readlines()
